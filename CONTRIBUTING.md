# Contributing to Upstream

First off, thank you for considering contributing to Upstream! We appreciate your time and effort in helping make simple and open logging better for everyone.

To ensure a smooth process for both contributors and maintainers, please review the guidelines below before getting started.

---

## 🏗️ Where You Can Contribute

Upstream operates on a mixed-source model. Please note where contributions are currently accepted:

### ✅ `upstream-sdk` & Marketing Site
We actively welcome code contributions, bug fixes, documentation improvements, and typo corrections to both the SDK and the public-facing marketing site.

### 🚫 Upstream Core (The App)
Upstream Core is currently closed-source while we continue active development. **We are not accepting code contributions for the backend app at this time.** However, we still want your ideas! If you have a feature suggestion or found a bug in the core app, please [open an issue](#-reporting-bugs--requesting-features) and describe what you'd like to see.

---

## 🛑 The Golden Rule: Open an Issue First

Before you write any code or open a Pull Request, **please open an issue first**.

Whether you are fixing a bug or adding a new feature to the SDK, discussing it with us beforehand ensures that your proposed changes align with the project's roadmap and saves you from doing work that might not get merged.

---

## 🐛 Reporting Bugs & Requesting Features

We use GitHub Issues to track bugs and feature requests.

* **Bug Reports:** Please provide a clear description of the issue, the version of the SDK you are using, and steps to reproduce the behavior.
* **Feature Requests:** Explain what the feature is, why you need it, and how it should work.

---

## 🛠️ Pull Request Process

Once your issue has been discussed and approved, you can start working on your PR!

1. **Fork the repository** and create your branch from `main`.
2. **Write clear, concise commit messages.**
3. **Test your changes** to ensure you haven't broken any existing functionality. If you are adding a new feature to the SDK, please update the relevant documentation.
4. **Open a Pull Request** and reference the original issue number in your PR description (e.g., `Closes #123`).
5. **Wait for review.** A maintainer will review your code, provide feedback, and merge it once approved.
